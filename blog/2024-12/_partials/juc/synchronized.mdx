> `synchronized` 是 Java 基于操作系统 **Mutex Lock** 的提供的**可重入**的**非公平**锁

- 作用在不同位置

  - 锁一个对象

  ```java
  private final Object lock = new Object();

  synchronized (lock) {
    // 代码块
  }

  synchronized (this) {
    // 代码块
  }

  ```

  - 锁整个类

  ```java
  synchronized (Class.class) {
    // 代码块
  }
  ```

  - 非静态方法上声明的形式锁的是**当前对象**

  ```java
  public synchronized void method() {
    // 代码块
  }

  ```

  - 静态方法上声明的形式锁的是**类**

  ```java
  public static synchronized void method() {
    // 代码块
  }
  ```

- 锁升级

  1. **无锁转换为偏向锁**

     - 通过 `hashCode` 判断锁对象是否为无锁
     - 如果为 **无锁**，则将锁对象的 `unused` 以及 `hashCode` 设置为当前线程（操作系统级别）的 `ID` 通过 `epoch` 记录时间，并将锁标记设置为 **偏向锁** - `1 01`。

  2. **偏向锁转换为轻量级锁**

     - 判断当前共享资源是否为 **偏向锁**，如果是，将锁对象的 `epoch` 与当前线程 `Thread ID` 进行比较。
       - 如果相同，则直接获取到 **偏向锁**。
       - 如果不同，则通过 `CAS` 操作将锁对象的 `epoch` 加 1，并将偏向锁标志为的状态设置为**轻量级锁** - `00`。
     - 通过 `CAS` 操作获取 **轻量级锁**。失败则自旋等待。

  3. **轻量级锁转换为重量级锁**

     - 通过 `CAS` 操作获取 **轻量级锁** _失败一定次数_，或者*存在自旋等待的线程*，则将锁对象的状态设置为 **重量级锁** - `10`。
     - 此时线程获取锁会阻塞，底层通过操作系统 :term[Mutex Lock]{./terms/java#mutex-lock} 实现。
